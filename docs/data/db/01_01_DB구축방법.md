### 1단계: Docker 설치

#### Windows에서 Docker 설치

1. **WSL 2 설치 및 활성화**:
   ```powershell
   # PowerShell을 관리자 권한으로 실행
   wsl --install

   # 재부팅 후 WSL 2를 기본값으로 설정
   wsl --set-default-version 2
   ```

2. **Docker Desktop 다운로드 및 설치**:
   - URL: https://www.docker.com/products/docker-desktop/
   - "Download for Windows" 클릭
   - 설치 파일 실행 (`Docker Desktop Installer.exe`)
   - 설치 옵션:
     - [✓] Use WSL 2 instead of Hyper-V (권장)
     - [✓] Add shortcut to desktop

3. **Docker Desktop 실행 및 확인**:
   - 시작 메뉴에서 "Docker Desktop" 실행
   - 하단 상태 바가 녹색(Running)으로 변경될 때까지 대기

4. **PowerShell/CMD에서 확인**:
   ```bash
   docker --version
   docker-compose --version
   ```

   **예상 출력**:
   ```
   Docker version 25.0.0, build xxx
   Docker Compose version v2.24.0
   ```

#### Mac에서 Docker 설치

1. **Docker Desktop 다운로드**:
   - URL: https://www.docker.com/products/docker-desktop/
   - Mac 칩에 따라 선택:
     - **Intel Mac**: "Mac with Intel chip" 다운로드
     - **Apple Silicon (M1/M2/M3)**: "Mac with Apple chip" 다운로드

2. **설치**:
   - `.dmg` 파일 다운로드 후 실행
   - Docker.app을 Applications 폴더로 드래그
   - Applications에서 Docker 실행
   - 권한 요청 시 승인

3. **터미널에서 확인**:

#### 2.2 docker-compose.yml 작성

프로젝트 디렉토리에 `docker-compose.yml` 파일을 생성합니다:

```yaml
version: '3.8'

services:
  postgres:
    image: pgvector/pgvector:pg17
    container_name: ddoksori_db
    restart: unless-stopped

    # 환경 변수
    environment:
      POSTGRES_DB: ${POSTGRES_DB:-ddoksori}
      POSTGRES_USER: ${POSTGRES_USER:-postgres}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
      POSTGRES_INITDB_ARGS: "--encoding=UTF-8 --locale=C"
      TZ: Asia/Seoul

    # 포트 매핑
    ports:
      - "${POSTGRES_PORT:-5432}:5432"

    # 볼륨 마운트
    volumes:
      # PostgreSQL 데이터 영구 저장
      - postgres_data:/var/lib/postgresql/data

      # 초기화 스크립트 (최초 실행 시 자동 실행)
      - ./01_00_unified_schema.sql:/docker-entrypoint-initdb.d/01_schema.sql:ro

      # 추가 설정 파일 (선택사항)
      # - ./postgresql.conf:/etc/postgresql/postgresql.conf:ro

    # 헬스 체크
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER:-postgres} -d ${POSTGRES_DB:-ddoksori}"]
      interval: 10s
      timeout: 5s
      retries: 5

    # 리소스 제한 (선택사항)
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 4G
        reservations:
          memory: 2G

  # CloudBeaver (선택사항, 웹 기반 DB 관리 도구)
  # DBeaver 데스크톱 앱을 사용하는 경우 이 서비스는 불필요합니다

  #   container_name: ddoksori_cloudbeaver

  #
  #   ports:
  #     - "${CLOUDBEAVER_PORT:-8978}:8978"
  #
  #   volumes:
  #     - cloudbeaver_data:/opt/cloudbeaver/workspace
  #
  #   depends_on:
  #     postgres:
  #       condition: service_healthy

# 볼륨 정의
volumes:
  postgres_data:
    name: ddoksori_postgres_data
  # cloudbeaver_data:
  #   name: ddoksori_cloudbeaver_data
```

```python
import os

from dotenv import load_dotenv

load_dotenv()

try:
    conn = psycopg2.connect(
        host="localhost",
        port=os.getenv("POSTGRES_PORT", 5432),
        database=os.getenv("POSTGRES_DB"),
        user=os.getenv("POSTGRES_USER"),
        password=os.getenv("POSTGRES_PASSWORD")

    cursor = conn.cursor()
    cursor.execute("SELECT version();")
    version = cursor.fetchone()

    cursor.close()
    conn.close()

except Exception as e:
    print(f"❌ 오류: {e}")
```

### 🚨 Docker 크로스 플랫폼 주의사항

#### 1. 파일 경로 차이

**Windows (docker-compose.yml)**:
```yaml
volumes:
  - ./01_00_unified_schema.sql:/docker-entrypoint-initdb.d/01_schema.sql:ro
```

**Mac/Linux (docker-compose.yml)**:
```yaml
# 동일하게 사용 가능 (상대 경로 권장)
volumes:
  - ./01_00_unified_schema.sql:/docker-entrypoint-initdb.d/01_schema.sql:ro
```

**권장**: 상대 경로(`./ 시작`)를 사용하면 모든 OS에서 동일하게 작동합니다.

#### 2. 줄바꿈 문자 (Line Endings)

**문제**: Windows는 CRLF(`\r\n`), Mac/Linux는 LF(`\n`)를 사용합니다.

**해결**:
- Git 설정으로 자동 변환:
  ```bash
  # Windows
  git config --global core.autocrlf true

  # Mac/Linux
  git config --global core.autocrlf input
  ```

- `.gitattributes` 파일 생성:
  ```
  *.sql text eol=lf
  *.sh text eol=lf
  *.yml text eol=lf
  ```

#### 3. Apple Silicon (M1/M2/M3) 주의사항

**pgvector 이미지**: `pgvector/pgvector:pg17`은 **arm64 아키텍처를 지원**합니다.

**확인**:
```bash
# Mac에서 아키텍처 확인
uname -m

# arm64 또는 x86_64 출력
```

**만약 arm64에서 문제 발생 시**:
```yaml
# docker-compose.yml에 platform 명시
services:
  postgres:
    image: pgvector/pgvector:pg17
    platform: linux/amd64  # 또는 linux/arm64
```

#### 4. 볼륨 성능

**Windows (WSL 2)**:
- ✅ **권장**: WSL 2 파일 시스템 내부에 프로젝트 배치
  - 예: `/home/username/project/...`
  - 성능: ~10배 빠름
- ❌ **비권장**: Windows 파일 시스템 (`/mnt/c/Users/...`)
  - 성능: 느림 (파일 시스템 변환 오버헤드)

**Mac**:
- ✅ gRPC FUSE 사용 (Docker Desktop 설정)

```sql
DECLARE
    test_vector vector(1536);

    -- 0으로 채워진 1536차원 벡터 생성

    INSERT INTO vector_chunks

    RAISE NOTICE '✅ 테스트 데이터 삽입 성공!';
END $$;

-- 확인
SELECT chunk_id, dataset_type, text, category
FROM vector_chunks
WHERE chunk_id = 'test_001';

-- 벡터 검색 테스트
SELECT chunk_id, 1 - (embedding <=> array_fill(0.0, ARRAY[1536])::vector(1536)) AS similarity
FROM vector_chunks

DELETE FROM vector_chunks WHERE chunk_id = 'test_001';
```
