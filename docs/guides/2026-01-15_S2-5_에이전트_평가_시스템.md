# S2-5: 에이전트 평가 기준 수립 및 평가 시스템 구현

**날짜**: 2026-01-15

## 개요

똑소리 멀티 에이전트 시스템(MAS)의 성능을 정량적으로 측정하기 위한 평가 시스템을 구현했습니다. Golden Set 기반의 CLI 평가 도구를 통해 각 에이전트의 품질을 지속적으로 모니터링할 수 있습니다.

## 변경 사항

### 1. Golden Set 구축 (280 샘플)

| 파일 | 샘플 수 | 용도 |
|------|---------|------|
| `query_analysis.jsonl` | 110개 | 질의 분석 에이전트 평가 |
| `retrieval.jsonl` | 60개 | 정보 검색 에이전트 평가 |
| `review.jsonl` | 110개 | 법률 검토 에이전트 평가 |

**위치**: `backend/data/golden_set/`

### 2. 평가 메트릭 클래스

#### QueryAnalysisMetrics (`backend/rag/evaluation/query_analysis_metrics.py`)
- 질의 유형 분류 정확도
- 키워드 추출 정밀도/재현율
- 기관 힌트 정확도

#### ReviewMetrics (`backend/rag/evaluation/review_metrics.py`)
- 위반 탐지 정밀도/재현율
- 오탐률 (False Positive Rate)
- 이진 분류 정확도

### 3. CLI 평가 스크립트

| 스크립트 | 대상 에이전트 |
|----------|--------------|
| `evaluate_query_analysis.py` | 질의 분석 에이전트 |
| `evaluate_review.py` | 법률 검토 에이전트 |
| `generate_report.py` | 통합 리포트 생성 |

**위치**: `backend/scripts/evaluation/`

## 사용 방법

### 환경 설정

```bash
conda activate dsr
cd backend
```

### 질의 분석 에이전트 평가

```bash
python -m scripts.evaluation.evaluate_query_analysis \
  --golden-set ./data/golden_set/query_analysis.jsonl \
  --output ./results/qa_eval.json
```

### 법률 검토 에이전트 평가

```bash
python -m scripts.evaluation.evaluate_review \
  --golden-set ./data/golden_set/review.jsonl \
  --output ./results/review_eval.json
```

### 정보 검색 에이전트 평가 (DB 연결 필요)

```bash
python -m scripts.evaluation.run_evaluation \
  --dataset ./data/golden_set/retrieval.jsonl \
  --output ./results/retrieval_eval.json
```

### 통합 리포트 생성

```bash
python -m scripts.evaluation.generate_report \
  --results-dir ./results \
  --output ./reports/eval_report.csv \
  --format both
```

## Golden Set 형식

### query_analysis.jsonl

```json
{
  "id": "qa_001",
  "query": "소비자보호법 제17조 환불 규정",
  "expected_query_type": "law",
  "expected_keywords": ["소비자보호법", "17조", "환불"],
  "expected_agency_hint": "KCA",
  "expected_missing_fields": [],
  "category": "법령조회"
}
```

### retrieval.jsonl

```json
{
  "id": "ret_001",
  "query": "에어컨 환불 요청",
  "expected_contexts": [
    {"doc_type": "dispute", "doc_id": "KCA-001", "relevance": "essential"},
    {"doc_type": "law", "doc_id": "consumer-law-7", "relevance": "supporting"}
  ],
  "expected_agency": "KCA",
  "category": "가전제품_하자"
}
```

### review.jsonl

```json
{
  "id": "rev_001",
  "answer_text": "반드시 환불받으실 수 있습니다.",
  "expected_violations": [
    {"pattern": "반드시", "type": "absolute_expression"}
  ],
  "is_violation": true
}
```

## 향후 개선 사항

1. **키워드 추출 개선**: 현재 규칙 기반 → LLM 보조 또는 NER 모델 적용 검토
2. **기관 힌트 정확도 향상**: 키워드 매칭 규칙 확장 필요
3. **생성 에이전트 평가**: RAGAS 기반 Faithfulness, Answer Relevancy 지표 추가 (LLM API 비용 고려)
