## 2. 기대 효과

| 항목 | 기존 (2-way RRF) | 개선 (3-way RRF) |
|:-----|:---------------:|:----------------:|
| 검색 소스 | Dense + Lexical | Dense + Lexical + Sparse |
| 한국어 키워드 매칭 | FTS 의존 | BGE-M3 Sparse 보완 |
| Recall 예상 | 기준 | +5~15% 향상 (검증 필요) |

### RRF 가중치 조절

```
score(d) = w_dense × 1/(k+rank_dense)
         + w_lexical × 1/(k+rank_lexical)
         + w_sparse × 1/(k+rank_sparse)
```

- 환경변수로 A/B 테스트 가능
- 기본값: 1:1:1 동등 가중치

## 3. 테스트 방법

### 3.1 BGE-M3 서버 테스트

```bash
# BGE-M3 서버 시작
docker-compose up -d bge_m3_embedding

# 헬스체크
curl http://localhost:8003/health

# 임베딩 테스트
curl -X POST http://localhost:8003/embed \
  -H "Content-Type: application/json" \
  -d '{"text": "헬스장 환불 규정", "return_dense": true, "return_sparse": true}'
```

### 3.2 배치 임베딩 실행

```bash
# DB 마이그레이션

  < backend/database/migrations/003_add_bge_m3_columns.sql

# 배치 임베딩 (소량 테스트)
python backend/scripts/data_loading/embed_bge_m3_all_data.py --limit 100

# 전체 임베딩 (74,000개, GPU 권장)
python backend/scripts/data_loading/embed_bge_m3_all_data.py
```

### 3.4 검색 품질 비교

```bash
# 2-way vs 3-way 비교 테스트
PYTHONPATH=. python -c "
from rag.hybrid_retriever import HybridRetriever
import os

DB_CONFIG = {
    'host': os.getenv('DB_HOST', 'localhost'),
    'port': 5432,
    'database': 'ddoksori',
    'user': 'postgres',
    'password': 'postgres'
}

retriever = HybridRetriever(DB_CONFIG, enable_sparse=True)
retriever.connect()

query = '헬스장 환불 규정'

# 3-way
retriever.enable_sparse = True
results_3way = retriever.search(query, top_k=5)

# 2-way
retriever.enable_sparse = False
results_2way = retriever.search(query, top_k=5)

print('=== 3-way RRF ===')
for r in results_3way:
    print(f'  {r.chunk_id}: {r.similarity:.4f}')

print('=== 2-way RRF ===')
for r in results_2way:
    print(f'  {r.chunk_id}: {r.similarity:.4f}')

retriever.close()
"
```

## 4. 환경 변수

`backend/.env`에 추가:

```bash
# BGE-M3 임베딩 설정
BGE_M3_REMOTE_URL=              # 원격 서버 URL (선택)
BGE_M3_LOCAL_PORT=8003          # 로컬 포트

# 임베딩 모델 선택
EMBEDDING_MODEL=kure-v1         # kure-v1 | bge-m3

# Sparse 검색 활성화 (3-way RRF)
ENABLE_SPARSE_SEARCH=false      # true | false

# RRF 가중치 (A/B 테스트용)
RRF_WEIGHT_DENSE=1.0
RRF_WEIGHT_LEXICAL=1.0
RRF_WEIGHT_SPARSE=1.0
```
