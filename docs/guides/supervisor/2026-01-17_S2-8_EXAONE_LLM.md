# S2-8 EXAONE 3.5 2.4B LLM 통합

> **작성일**: 2026-01-17
> **PR**: S2-8
> **핵심**: ReAct 추론 노드에 EXAONE LLM 기반 추론 추가

---

## 1. 변경 사항

### 신규 파일

| 파일 | 역할 |
|:-----|:-----|
| `backend/app/llm/__init__.py` | LLM 모듈 초기화 |
| `backend/app/llm/exaone_client.py` | EXAONE 3.5 2.4B 클라이언트 (RunPod vLLM) |
| `backend/scripts/testing/orchestrator/test_react_llm.py` | LLM 통합 테스트 |

### 수정 파일

| 파일 | 변경 내용 |
|:-----|:---------|
| `backend/app/orchestrator/nodes/react_think.py` | LLM 기반 추론 추가 (`_llm_based_think`) |
| `backend/.env.example` | EXAONE 환경 변수 추가 |

### 핵심 기능

1. **ExaoneLLMClient**
   - RunPod vLLM 서버와 OpenAI-compatible API로 통신
   - 헬스체크 및 가용성 캐싱
   - `LLMUnavailableError` 예외 처리

2. **LLM 기반 추론** (`REACT_THINK_MODE=llm`)
   - EXAONE 3.5 2.4B로 상황 분석 및 액션 결정
   - JSON 형식 응답 파싱
   - 실패 시 규칙 기반 폴백 자동 전환

## 4. 전체 실행 가이드

### 4.1 환경 변수 설정

`backend/.env`에 추가:

```bash
# EXAONE 3.5 2.4B LLM 설정
EXAONE_RUNPOD_URL=https://<pod-id>-8000.proxy.runpod.net/v1
EXAONE_RUNPOD_API_KEY=dummy
EXAONE_MODEL=LGAI-EXAONE/EXAONE-3.5-2.4B-Instruct
EXAONE_TEMPERATURE=0.1
EXAONE_MAX_TOKENS=512
EXAONE_TIMEOUT=10

# 추론 모드 선택
REACT_THINK_MODE=rule   # llm | rule
```

### 4.2 모드 전환

```bash
# 규칙 기반 모드 (기본값, RunPod 불필요)
export REACT_THINK_MODE=rule

# LLM 기반 모드 (RunPod 필요)
export REACT_THINK_MODE=llm
export EXAONE_RUNPOD_URL=https://<pod-id>-8000.proxy.runpod.net/v1
```

### 4.3 RunPod vLLM 서버 배포 (선택)

**Pod 설정:**
- GPU: RTX 3090 (24GB) 이상
- Container Disk: 50GB
- Expose Port: 8000

**Pod 내 실행:**
```bash
pip install vllm

nohup python -m vllm.entrypoints.openai.api_server \
    --model LGAI-EXAONE/EXAONE-3.5-2.4B-Instruct \
    --host 0.0.0.0 \
    --port 8000 \
    --dtype auto \
    --max-model-len 4096 \
    --trust-remote-code \
    > /workspace/vllm.log 2>&1 &

# 헬스체크
curl http://localhost:8000/health
```

### 4.4 서버 실행

```bash
cd /home/maroco/LLM/backend
conda activate dsr
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```
