## 2026-02-06: OAuth 로그인 토큰 전달 방식 수정

### 📋 수정 파일
- `backend/app/api/auth.py`

### 🔧 수정 내용
OAuth 로그인 콜백 시 JWT 토큰을 프론트엔드로 전달하는 방식을 **Query Parameter**에서 **Hash Fragment**로 변경

#### 변경 전:
```python
redirect_url = (
    f"{config.frontend_url}/auth/callback?{urlencode(redirect_params)}"
)
```
- URL 형식: `http://localhost:5173/auth/callback?access_token=abc123`

#### 변경 후:
```python
redirect_url = (
    f"{config.frontend_url}/auth/callback#{urlencode(redirect_params)}"
)
```
- URL 형식: `http://localhost:5173/auth/callback#access_token=abc123`

### 🎯 수정 이유

#### 1. 프론트엔드 코드와의 호환성
- `AuthCallback.tsx`가 이미 Hash Fragment(`#`) 방식으로 토큰을 읽도록 구현되어 있음
- `parseHashParams(window.location.hash)` 사용

#### 2. 보안 강화
- **Query Parameter (`?`) 문제점:**
  - 서버 로그(nginx, Apache)에 토큰이 기록됨
  - Referer 헤더에 토큰이 포함될 수 있음
  - 서버 관리자가 로그에서 토큰을 볼 수 있음

- **Hash Fragment (`#`) 장점:**
  - `#` 뒤의 내용은 서버로 전송되지 않음
  - 브라우저에서만 JavaScript로 접근 가능
  - 서버 로그에 토큰이 남지 않아 보안성 향상
  - Referer 헤더에 포함되지 않음

### 📊 영향 범위
- ✅ Google OAuth 로그인 콜백
- ✅ Naver OAuth 로그인 콜백
- ❌ 관리자 로그인 (영향 없음, 별도 시스템)

### ✅ 검증 필요
1. Google 로그인 테스트

4. 로그인 후 채팅 세션 로드 확인

### 🔧 수정 2: Nginx 프로덕션 프록시 설정

#### 파일: `frontend/nginx.conf`

#### 변경 전:
```nginx
location /auth {
    proxy_pass http://backend:8000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```
- 모든 `/auth/*` 경로를 백엔드로 프록시

#### 변경 후:
```nginx
# OAuth backend endpoints only (exclude /auth/callback - frontend route)
location ~ ^/auth/(google|naver|me|verify|delete-account) {
    proxy_pass http://backend:8000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```
- 정규식 `~`를 사용하여 백엔드 API 경로만 매칭
- `/auth/callback`은 매칭되지 않아 SPA routing (62-64번 줄)으로 처리됨

```typescript
function getHiddenSessions(userId: string | null): Set<string> {
  const key = getUserHiddenSessionsKey(userId);
  const hidden = storage.get<string[]>(key, false) || [];
  return new Set(hidden);
}

// 세션을 숨긴 목록에 추가
function addHiddenSession(userId: string | null, sessionId: string): void {
  const key = getUserHiddenSessionsKey(userId);
  const hidden = getHiddenSessions(userId);
  hidden.add(sessionId);
  storage.set(key, Array.from(hidden), false);
}

// 세션 목록에서 숨긴 세션 필터링
function filterHiddenSessions(sessions: ChatSession[], userId: string | null): ChatSession[] {
  const hiddenSessions = getHiddenSessions(userId);
  return sessions.filter(session => !hiddenSessions.has(session.id));
}
```

### 🔧 수정 2: 세션 Title 자동 생성 개선

#### 문제점
- 백엔드에서 "dispute 상담", "general 상담" 기본값만 저장
- 분쟁 상담 시 `[분쟁 정보]...` 폼 데이터가 title로 표시됨

#### 해결 방법

**1. Title 생성 로직 함수화**
```typescript
// frontend/src/features/chat/chat.store.ts
function generateTitleFromMessages(type: ChatType, messages: MessageWithCitations[]): string {
  if (type === 'dispute') {
    // [분쟁 정보]로 시작하지 않는 실제 질문 찾기
    const actualQuestion = messages.find(
      (msg) => msg.type === 'user' && !msg.content.startsWith('[분쟁 정보]')
    );

    if (actualQuestion) {
      return actualQuestion.content.substring(0, 30) + '...';
    } else {
      // 폼 데이터에서 구매품목 추출
      const formMessage = messages.find((msg) => msg.content.includes('구매품목'));
      if (formMessage) {
        const match = formMessage.content.match(/구매품목\s*[:：]\s*([^\n]+)/);
        return match ? `${match[1].trim()} 관련 상담` : '분쟁 상담';
      }
      return '분쟁 상담';
    }
  } else {
    // 일반 상담: 첫 번째 user 메시지 사용
    const userMessage = messages.find((msg) => msg.type === 'user');
    return userMessage
      ? userMessage.content.substring(0, 30) + '...'
      : '일반 상담';
  }
}
```

**2. syncWithBackend에서 title 재생성**
```typescript
// 백엔드에서 기본값이면 메시지에서 재생성
const needsTitleRegeneration =
  backendSession.title === 'dispute 상담' ||
  backendSession.title === 'general 상담' ||
  !backendSession.title;

const finalTitle = needsTitleRegeneration
  ? generateTitleFromMessages(sessionType, messages)
  : backendSession.title;
```

### 🔧 수정 4: 백엔드 session_id 우선 사용 및 동기화

#### 해결 방법

**1. 세션 ID 우선순위 설정**
```typescript
// frontend/src/features/chat/chat.store.ts
let newSessionId = state.backendSessionId || state.currentSessionId;
```

**2. 임시 ID → 백엔드 ID 자동 변환**
```typescript
if (state.backendSessionId && state.currentSessionId &&
    state.backendSessionId !== state.currentSessionId) {

  // 이미 백엔드 ID를 가진 세션이 있는지 확인 (중복 방지)
  const backendSessionExists = sessions.some((s) => s.id === state.backendSessionId);

  if (!backendSessionExists) {
    const oldSessionIndex = sessions.findIndex((s) => s.id === state.currentSessionId);
```

### 🔧 수정 5: 페이지 Focus 시 자동 동기화 (멀티 디바이스)

#### 기능 설명
브라우저 탭으로 돌아왔을 때 자동으로 백엔드에서 최신 데이터를 가져와 다른 디바이스의 변경사항을 반영

#### 구현 내용

```typescript
// frontend/src/app/RootLayout.tsx
useEffect(() => {
  if (!isLoggedIn || !token) return;

  const handleVisibilityChange = async () => {
    if (document.visibilityState === 'visible') {
      const now = Date.now();
      const timeSinceLastSync = now - lastSyncTime.current;

      // 최소 30초 간격으로 동기화 (너무 자주 호출 방지)
      if (timeSinceLastSync > 30000) {
        console.log('[RootLayout] Page became visible, syncing with backend...');
        await syncWithBackend(token);
        lastSyncTime.current = now;
        console.log('[RootLayout] Sync complete');
      } else {
        console.log(`[RootLayout] Skipping sync (last sync was ${Math.floor(timeSinceLastSync / 1000)}s ago)`);
      }
    }
  };

  document.addEventListener('visibilitychange', handleVisibilityChange);
  return () => document.removeEventListener('visibilitychange', handleVisibilityChange);
}, [isLoggedIn, token, syncWithBackend]);
```

#### 동작 시나리오
1. **크롬 브라우저**: 새 질문 작성
2. **엣지 브라우저**: 다른 탭에서 엣지 탭으로 전환
3. **자동 동기화**: `visibilitychange` 이벤트 발생 → 백엔드에서 최신 데이터 가져옴
4. **사이드바 업데이트**: 크롬의 질문이 자동으로 표시됨

#### 최적화
- 30초 간격 제한으로 과도한 API 호출 방지
- 네트워크/배터리 사용량 최소화

```python
# 변경 후
MEMORY_POLICIES = {
    "general": MemoryPolicy(max_turns=10, compact_enabled=False, sliding_window=5),
}
```

**이유:**
- 멀티 디바이스 동기화를 위해 일반 상담도 DB에 저장 필요
- `max_turns=0`이면 DB 저장이 안되어 동기화 불가
