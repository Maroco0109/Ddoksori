## 1.2 상세 수정 내용

### 수정 A: HNSW 인덱스 파라미터 최적화

**현재 상태** (`01_00_unified_schema.sql:197-199`):
```sql
CREATE INDEX idx_embedding_hnsw ON vector_chunks
USING hnsw (embedding vector_cosine_ops)
WITH (m = 16, ef_construction = 64);  -- 기본값
```

**문제점**:
- `m=16, ef_construction=64`는 pgvector 기본값
- 52,000개 규모에서 최적화되지 않음
- Recall 손실 발생

**개선안**:
```sql
-- 기존 인덱스 삭제
DROP INDEX IF EXISTS idx_embedding_hnsw;

-- 최적화된 인덱스 재생성
CREATE INDEX idx_embedding_hnsw ON vector_chunks
USING hnsw (embedding vector_cosine_ops)
WITH (m = 24, ef_construction = 100);
```

**파라미터 설명**:
- `m = 24`: 각 노드 연결 수 증가 (16 → 24)
  - 검색 정확도 향상
  - 인덱스 크기 약 50% 증가
- `ef_construction = 100`: 구축 시 탐색 범위 증가 (64 → 100)
  - 인덱스 품질 향상
  - 구축 시간 약 60% 증가

**예상 효과**:
- Recall@10: +5~8%
- 인덱스 재구축 시간: 약 15-20분

**주의사항**:
- 인덱스 재구축 중 검색 성능 저하 가능
- 서비스 점검 시간에 실행 권장

# 추가할 함수
def determine_rrf_k(query: str, query_analysis: dict = None) -> int:
    """
    쿼리 유형에 따른 RRF k값 결정

    - k값 낮음 → BM25(키워드) 가중치 ↑
    - k값 높음 → Vector(의미) 가중치 ↑

    Returns:
        int: 40~80 범위의 k값
    """
    import re

    # 법령 직접 참조 패턴
    law_patterns = [
        r'제?\d+조',           # 제16조, 16조
        r'법\s*제?\d+',        # 법 제16
        r'시행령|시행규칙',
        r'소비자기본법|전자상거래법|할부거래법',
    ]

    # 법령 키워드 포함 시 → BM25 우선 (k 낮춤)
    for pattern in law_patterns:
        if re.search(pattern, query):
            return 40

    # query_analysis 기반 판단
    if query_analysis:
        mode = query_analysis.get("mode", "")
        query_type = query_analysis.get("query_type", "")

        if query_type == "law_direct":
            return 40
        elif query_type == "criteria":
            return 50
        elif mode in ["case_search", "general"]:
            return 80

    # 기본값
    return 60


# 기존 함수 수정

### 2.2.2 [1순위] 동적 유사도 임계값 + 중복 제거

**파일**: `app/agents/retrieval/tools/hybrid_retriever.py` 또는 `app/orchestrator/nodes/retrieval_merge.py`

**추가할 함수들**:
```python
def adaptive_similarity_threshold(
    results: List[Dict],
    min_results: int = 3,
    min_threshold: float = 0.4,
    max_threshold: float = 0.7
) -> float:
    """
    검색 결과 품질 기반 동적 임계값

    Returns:
        float: 0.4 ~ 0.7 범위의 임계값
    """
    if not results:
        return min_threshold

    similarities = [r.get('vector_similarity', r.get('similarity', 0))
                    for r in results[:10]]
    if not similarities:
        return min_threshold

    max_sim = max(similarities)

    # 최고 유사도의 70%를 기준
    dynamic_threshold = max_sim * 0.70

    # 최소 결과 수 보장
    sorted_results = sorted(results, key=lambda x: x.get('rrf_score', 0), reverse=True)
    if sum(1 for r in sorted_results
           if r.get('vector_similarity', 0) >= dynamic_threshold) < min_results:
        if len(sorted_results) >= min_results:
            return sorted_results[min_results - 1].get('vector_similarity', min_threshold) - 0.01

    return max(min(dynamic_threshold, max_threshold), min_threshold)


def deduplicate_results(results: List[Dict]) -> List[Dict]:
    """
    chunk_id 기반 중복 제거 (높은 점수 유지)

    Returns:
        List[Dict]: 중복 제거된 결과
    """
    seen = {}
    for r in results:
        chunk_id = r.get('chunk_id')
        if not chunk_id:
            continue

        score = r.get('rrf_score', r.get('similarity', 0))
        if chunk_id not in seen or score > seen[chunk_id].get('rrf_score', 0):
            seen[chunk_id] = r

    return list(seen.values())
```

### 2.2.3 [1순위] retrieval_merge 개선

**파일**: `app/orchestrator/nodes/retrieval_merge.py`

**수정 내용**:
```python
def retrieval_merge_node(state: ChatState) -> Dict:
    individual_results = state.get("individual_retrieval_results", [])

    # 1. 기존 병합 로직
    merged = _merge_to_retrieval_result(individual_results)

    # 2. 전체 문서 플래튼
    all_docs = []
    for section in ['laws', 'criteria', 'disputes', 'counsels']:
        all_docs.extend(merged.get(section, []))

    # 3. 중복 제거 (신규)
    deduped = deduplicate_results(all_docs)

    # 4. RRF 점수 기준 재정렬 (신규)
    sorted_docs = sorted(deduped, key=lambda x: x.get('rrf_score', 0), reverse=True)

    # 5. 동적 임계값 적용 (신규)
    threshold = adaptive_similarity_threshold(sorted_docs)
    filtered = [d for d in sorted_docs
                if d.get('vector_similarity', d.get('similarity', 0)) >= threshold]

    # 6. 섹션별 재분류
    final_merged = _redistribute_to_sections(filtered)

    return {"retrieval": final_merged}


def _redistribute_to_sections(docs: List[Dict]) -> Dict:
    """재정렬된 문서를 섹션별로 재분류"""
    result = {
        'laws': [],
        'criteria': [],
        'disputes': [],
        'counsels': [],
        'agency': {}
    }

    source_map = {
        'law': 'laws',
        'criteria': 'criteria',
        'case': 'disputes',
        'counsel': 'counsels',
    }

    for doc in docs:
        source = doc.get('source', doc.get('dataset_type', ''))
        section = source_map.get(source)
        if section:
            result[section].append(doc)

    return result
```

### 2.2.4 [2순위] Cross-Encoder 재랭킹 서비스

**파일**: `app/services/reranker.py` (신규 생성)

```python
"""
Cross-Encoder 기반 재랭킹 서비스

의존성: pip install sentence-transformers
"""

from typing import List, Dict, Optional
import asyncio
from functools import lru_cache

# 전역 인스턴스 (지연 로딩)
_reranker = None


def get_reranker():
    """싱글톤 재랭커 인스턴스"""
    global _reranker
    if _reranker is None:
        from sentence_transformers import CrossEncoder
        _reranker = CrossEncoder('cross-encoder/ms-marco-MiniLM-L-6-v2')
    return _reranker


async def rerank_results(
    query: str,
    results: List[Dict],
    top_n: int = 10,
    text_field: str = 'text'
) -> List[Dict]:
    """
    Cross-Encoder 기반 재랭킹

    Args:
        query: 사용자 쿼리
        results: 검색 결과 리스트
        top_n: 반환할 결과 수
        text_field: 텍스트 필드명

    Returns:
        재랭킹된 결과 (top_n개)
    """
    if len(results) <= top_n:
        return results

    reranker = get_reranker()

    # 쿼리-문서 쌍 생성 (텍스트 길이 제한)
    pairs = [
        (query, doc.get(text_field, '')[:512])
        for doc in results
    ]

    # 동기 함수를 비동기로 실행
    scores = await asyncio.to_thread(reranker.predict, pairs)

    # 점수 할당
    for i, doc in enumerate(results):
        doc['rerank_score'] = float(scores[i])

    # 재정렬
    reranked = sorted(results, key=lambda x: x['rerank_score'], reverse=True)

    return reranked[:top_n]
```

```python
def compress_context(
    results: List[Dict],
    max_tokens: int = 2000
) -> str:
    """
    Lost-in-the-middle 방지를 위한 컨텍스트 구성

    - 가장 관련성 높은 결과를 앞뒤에 배치
    - 토큰 제한 내에서 최대한 포함
    """
    if len(results) >= 3:
        # 1위, 마지막, 나머지 순서
        reordered = [results[0], results[-1]] + results[1:-1]
    else:
        reordered = results

    compressed = []
    current_tokens = 0

    for i, r in enumerate(reordered):
        text = r.get('text', '')
        # 대략적 토큰 추정 (4글자 = 1토큰)
        text_tokens = len(text) // 4

        if current_tokens + text_tokens > max_tokens:
            remaining = max_tokens - current_tokens
            text = text[:remaining * 4]

        source = r.get('chunk_id', r.get('source', 'unknown'))
        relevance = r.get('rerank_score', r.get('rrf_score', r.get('similarity', 0)))

        compressed.append(f"[문서 {i+1}] (관련도: {relevance:.2f})\n{text}\n출처: {source}")

        current_tokens += text_tokens
        if current_tokens >= max_tokens:
            break

    return "\n\n".join(compressed)
```
