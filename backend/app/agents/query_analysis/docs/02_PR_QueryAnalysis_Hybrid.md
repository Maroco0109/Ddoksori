# PR 2: Query Analysis Enhancement (Hybrid Intent & Synonyms)

## 1. 개요
현재 질의 분석(Query Analysis) 에이전트는 키워드 매칭에 크게 의존하고 있어, 사용자의 다양한 표현을 놓치거나(Recall 저하) 일반 질문을 분쟁으로 오분류하는 문제가 있습니다.
본 PR은 동의어 사전 확장, Hybrid 분류 로직 도입, Multi-Query Expansion을 통해 질의 분석의 정확도와 커버리지를 향상시킵니다.

## 2. 변경 대상 및 상세 계획

### 2.1. 동의어 사전 및 키워드 확장 (`backend/app/agents/query_analysis/agent.py`)
단순 리스트 형태의 키워드를 확장된 딕셔너리 형태로 개선하여 다양한 사용자 표현을 포괄합니다.

```python
# 기존: 단순 리스트
# DISPUTE_VERBS = ["환불", "반품", ...]

# 개선: 동의어 그룹핑
VERB_SYNONYMS = {
    "환불": ["환불", "반환", "취소", "청약철회", "돈 돌려받기", "환급", "반품", "결제 취소"],
    "교환": ["교환", "대체", "바꿈", "다른 제품으로", "교체", "변경"],
    "수리": ["수리", "고침", "AS", "애프터서비스", "보수", "고장", "수선"],
    "보상": ["보상", "배상", "물어내", "변상"],
}
```

### 2.2. Hybrid Intent Classification 도입
규칙 기반 분류의 한계를 보완하기 위해 LLM Fallback 로직을 정교화합니다.

1. **Layer 1 (Rule)**: 명확한 키워드(법률 용어, 특정 시스템 명령 등)가 있으면 즉시 분류.
2. **Layer 2 (LLM)**: 키워드가 없거나 모호한 경우(예: "이거 좀 이상해요"), LLM에게 의도 분류를 요청.
   - Prompt: "사용자의 발화가 구체적인 분쟁 해결 요청인지, 일반적인 정보 문의인지 분류하시오."

### 2.3. Multi-Query Expansion 구현
단일 쿼리만 생성하던 로직을 개선하여, 검색 확률을 높이는 다중 쿼리를 생성합니다.

```python
def _generate_search_queries(...):
    queries = [original_query]
    
    # 1. 동의어 치환 쿼리
    # 2. 법률 용어 결합 쿼리 ("노트북 환불" -> "노트북 환불 소비자분쟁해결기준")
    # 3. 사례 검색용 쿼리 ("... 조정 사례")
    
    return queries[:3] # 상위 3개 선정
```

## 3. 테스트 계획
1. **오분류 케이스 검증**: "환불이 뭔가요?" (General) vs "환불해 주세요" (Dispute) 구분 테스트.
2. **동의어 인식 검증**: "돈 돌려받고 싶어요" 입력 시 '환불' 키워드로 인식하고 관련 규정을 검색하는지 확인.
3. **검색 품질 확인**: Multi-Query 적용 전후 검색 결과(Recall) 비교 (로그 분석).

## 4. 기대 효과
- 질의 의도 분류 정확도 **15% 이상 향상** 예상.
- 다양한 사용자 표현(구어체, 동의어)에 대한 대응력 강화.
- 검색 실패(No Results) 비율 감소.
