## 법률/시행령
### 1) 상위 조문의 top-k 쏠림 문제
- `chunk_type`='호_분할'인 경우 윗 계층의 조, 항을 prior로 붙임
- `chunk_type`='항_분할'인 경우 윗 계층의 조와 첫 번째 항을 prior로 붙임
- 문제: 붙인 prior에서 강하게 매칭되면 해당 조문이 상위를 독식
- 해결: topk를 널널히 잡고, 받은 결과에 대해 같은 조를 기준으로 rrf_score 기준 최대 2개 제한.
    - 같은 조 판단 방법: `chunk_id`를 '_' 기준으로 split했을 때 앞 두 개가 무조건 '법 이름', '제n조' 정보이므로 이를 이용
        - 예시: `chunk_id`='소비자 보호에 관한 법률_제12조_제1호'일 때 '같은 조문'이란? "where `chunk_id` like '소비자 보호에 관한 법률_제12조%'"

### 2) 삭제 조문 문제
- `text`에 삭제된 조문이 '제735조()\n삭제 <2014. 3. 11.>' 이런 식으로 저장되어 있음.
- 해결: topk를 널널히 잡고, 받은 결과에 대해 `text`에서 '%()%삭제 <%' 같은 형식으로 되어 있는 검색 결과를 제거


## 별표/행정규칙
### '자식_청크', '손자_청크'에 '부모_청크'를 바로 조회하는 정보가 없는 문제
##### 현재 chunk_id 네이밍 규칙
- 부모-자식-손자 없을 때: `dispute3_단순`, chunk_type=’자식_청크’ 무조건
- 있을 때: `dispute3_부모`, `dispute_조건1`, `dispute_조건1_하위1` (숫자 무조건)

### 해결: 부모 청크 조회 방법
##### 자식 청크 -> 부모 청크
- (1) chunk_id맨 오른쪽이 `조건n`이면: `조건n` 떼고 `부모` 붙여서 조회
- (2) chunk_id 맨 오른쪽이 `조건n`이 아니면 조회 x
    - 예시1: `chunk_id`='별표2_1_I상품재화_5공산품34개품목_①전자제품사무용기기22_dispute5_조건1'
            => 부모 청크 있음,
            `chunk_id`='별표2_1_I상품재화_5공산품34개품목_①전자제품사무용기기22_dispute5_부모'로 조회
    - 예시2: `chunk_id`='별표2_1_I상품재화_5공산품34개품목_⑩연탄_dispute4_단순'
            => 부모 청크 없음,
            조회 X 

##### 손자 청크 -> 부모 청크
- 손자 청크의 부모 찾기: chunk_id에서 맨 오른쪽 `조건n_하위n` 떼고 `부모` 붙여서 조회
    - 예시: `chunk_id`='별표2_2_II서비스업_12개인및기타서비스업관련_⑯세탁업품목별내용연수_dispute11_조건1_하위3'
            => 부모 청크 있음,
            `chunk_id`='별표2_2_II서비스업_12개인및기타서비스업관련_⑯세탁업품목별내용연수_dispute11_부모'로 조회
